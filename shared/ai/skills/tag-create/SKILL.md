---
name: tag-create
description: git のアノテーション付きタグを作成して push する
---

## 手順

1. `git tag` で既存タグを確認し、最新のバージョンを把握する
2. タグ名を決定する
  - `/tag-create v1.2.3` のように引数が渡された場合はそれを使う
  - 引数がない場合は既存タグから次のバージョンを提案し、ユーザーに確認する
3. `git log --oneline <前タグ>..HEAD` で含まれるコミットを確認する
4. タグメッセージを作成する（コミット内容を要約した1〜3行）
5. ユーザーにタグ名・メッセージ・対象コミットを提示して確認を求める
6. `git tag -a <tag> -m "<message>"` でタグを作成する
7. `git push origin <tag>` でリモートに push する

## Tag Name Format

- セマンティックバージョニング: `vMAJOR.MINOR.PATCH`
- 例: `v1.0.0`, `v1.2.3`, `v2.0.0`

## バージョン番号の上げ方

| 変更内容 | 上げる番号 | 例 |
|---------|-----------|-----|
| 後方互換性のない変更 | MAJOR | v1.x.x → v2.0.0 |
| 後方互換性のある新機能 | MINOR | v1.1.x → v1.2.0 |
| バグ修正・軽微な改善 | PATCH | v1.1.1 → v1.1.2 |

## Constraints

- タグ push は取り消しが難しいため、必ずユーザーの確認を取ってから実行する
- 既存のタグと重複するタグ名は使わない
- main ブランチまたはリリース対象ブランチにいることを確認する
- 未 push のコミットがある場合は先に push するよう促す
